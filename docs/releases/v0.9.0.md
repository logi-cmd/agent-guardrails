# Release Notes - v0.9.0

**Release Date:** 2026-04-01

## What's New

**Enforce Command** — Injects guardrail instructions directly into each agent's system-level auto-read files. This is now the **recommended way** to ensure AI agents follow guardrail rules.

Previously, AGENTS.md rules were just "file content" that AI might or might not follow. With `enforce`, the guardrail check instruction is injected directly into each agent's system-level instruction file (CLAUDE.md, GEMINI.md, etc.), which is automatically read at a much higher priority.

## New Commands

### `enforce`

Injects guardrail check instructions into agent auto-read files:

```bash
# Enforce for all supported agents
agent-guardrails enforce --all

# Enforce for a specific agent
agent-guardrails enforce --agent claude-code

# Idempotent — safe to run multiple times
```

### `unenforce`

Safely removes enforced instructions while preserving any user-written content:

```bash
# Remove all enforced instructions
agent-guardrails unenforce --all

# Remove for a specific agent
agent-guardrails unenforce --agent claude-code
```

## Key Features

- **System-level priority**: Injected into files that agents read automatically as instructions
- **Bilingual templates**: English and Chinese for all 8 agents
- **Idempotent**: Running `enforce` twice is safe — detects already-enforced files and skips
- **Safe removal**: `unenforce` strips only the enforced section, preserving user content
- **Append mode**: For files with existing user content (CLAUDE.md, GEMINI.md, OPENCLAW.md), appends rather than overwrites

## Recommended Workflow

```bash
# 1. Setup
agent-guardrails setup --agent claude-code

# 2. Enforce (recommended)
agent-guardrails enforce --all

# 3. Work normally — AI will auto-run checks

# 4. Remove if needed
agent-guardrails unenforce --all
```
