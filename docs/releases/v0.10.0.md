# Release Notes - v0.10.0

**Release Date:** 2026-04-02

## Breaking Change: Agent Support Narrowed to 5

Removed 3 agents from the supported list. Going forward, `agent-guardrails` supports **5 agents**: claude-code, cursor, opencode, codex, gemini.

| Removed Agent | Why |
|---------------|-----|
| Windsurf | Text-only enforcement. No runtime interception capability. |
| OpenHands | Requires Docker/sandbox runtime. Mismatch with C-end user workflow. |
| OpenClaw | Load-time skills only. No runtime hooks to enforce guardrails. |

These agents could not reliably enforce guardrail checks at the system level, which defeats the purpose of `enforce`.

## What Changed

- **supportedAdapters reduced from 8 to 5** across all source files
- Removed agent entries (templates, locale files, enforce targets) for Windsurf, OpenHands, OpenClaw
- Deleted orphan hook scripts that only applied to removed agents
- Added Gemini adapter template in zh-CN
- Rewrote OpenCode plugin to use named export
- Added `installPluginFiles` step in setup flow

## Migration

If you were using any of the removed agents, run:

```bash
agent-guardrails unenforce --all
agent-guardrails enforce --all
```

This clears stale enforcement files and re-injects only for the 5 supported agents.

No action needed if you only use claude-code, cursor, opencode, codex, or gemini.
