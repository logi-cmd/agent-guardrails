## OSS Core

### Positioning line

**Everything you need to use `agent-guardrails` as a real merge gate.**

### Included

- `init / setup / plan / check`
- task contracts
- `check --review`
- `check --json`
- baseline runtime-backed workflow
- baseline scope / validation / continuity / performance output
- MCP server and agent adapters
- demos, docs, benchmarks, and CI baseline

### Who it is for

- first-time evaluators
- OSS users
- developers who want a trustworthy baseline before paying for anything

## Pro Local

### Positioning line

**For developers who already use AI coding every day and want less overhead per change.**

### Included promise

You are not paying for more code generation.

You are paying for:

- lower setup friction
- lower review effort
- stronger repo-specific guidance
- lower maintenance drift over time
- a workflow that beats a DIY plugin stack within the first week

### Upgrade triggers

Upgrade to Pro Local when:

- you use AI coding every day
- you are tired of writing the same guardrail setup over and over
- you want stronger signal without manually stitching tools together
- you want to keep your current coding agent but ship with more confidence

---

## Pro Cloud

### Positioning line

**For teams that need shared trust, not just solo confidence.**

### Included promise

- shared policies
- centralized review visibility
- historical analysis and auditability
- team-level workflows and governance

### Upgrade triggers

Upgrade to Pro Cloud when:

- multiple developers touch the same repos
- trust rules need to stay consistent across a team
- leadership wants visibility beyond one developer's terminal

## Comparison copy

### Why free first matters

The product earns trust by giving away the actual merge gate.

Paid tiers should never feel like:

> "The baseline only works after upgrade."

They should feel like:

> "The baseline already works. The upgrade removes friction and adds depth."

---

## FAQ snippet for pricing page

### Why not charge more?

Because the first paid buyer is usually a solo developer or a very small team already paying for AI coding.

The price has to feel obviously cheaper than the review time, cleanup time, and repo drift it saves.

### Why no price listed yet?

Pricing will be set after the first Pro Local features are real and shippable.

We would rather show you something worth paying for than ask you to pay for a promise.

If you want early access or want to shape what Pro Local includes first, [let us know](https://github.com/your-org/agent-guardrails/issues).

### Why not make semantic checks OSS-only or Pro-only?

Because OSS should remain a real merge gate.

Paid value should come from **efficiency and depth**, not from crippling the baseline.
