# Agent Rules

## Read First

1. `AGENTS.md`
2. `docs/PROJECT_STATE.md`
3. `README.md`
4. the files you intend to change

## Working Style

- Keep changes bounded to `src/` unless a task explicitly widens scope.
- Prefer the existing app shape and naming.
- When behavior changes, update the test alongside the implementation.
- Surface missing context instead of inventing it.

## Definition Of Done

- The change stays inside the allowed paths for the task.
- The demo checks still pass.
- The project state stays accurate.
